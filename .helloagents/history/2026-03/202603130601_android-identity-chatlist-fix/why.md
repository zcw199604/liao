# Android 身份与会话列表修复提案

## 1. 背景

当前 Android 客户端骨架已具备登录、身份、会话列表与聊天主链路，但身份页仍缺少编辑 / 删除闭环，会话列表也缺少显式空态、显式错误态与明确的全局收藏入口，占用后续联调与验收时间。

## 2. 目标

- 在不新增后端接口的前提下，复用现有 `updateIdentity` 与 `deleteIdentity` API，补齐身份页 Repository / ViewModel / UI 的编辑、删除最小闭环。
- 为会话列表补齐显式空态、显式错误态，并提供一个真实可点击的“全局收藏入口”占位按钮，保证代码中存在明确入口。
- 所有新增说明与注释保持简体中文，不回退现有同事已完成的 Android 改动。

## 3. 范围

### 3.1 范围内
- `IdentityFeature.kt`：补齐编辑 / 删除逻辑、删除确认、消息反馈与当前会话同步。
- `ChatListFeature.kt`：补齐显式空态、显式错误态与全局收藏入口占位按钮。
- Android 客户端知识库同步更新。

### 3.2 范围外
- 新增独立“全局收藏页”完整功能。
- 改造后端身份接口签名。
- 引入新的 Android 模块拆分或导航结构调整。

## 4. 验收标准

- 身份页可以创建、快速创建、编辑、删除、选择身份；编辑 / 删除当前身份时，本地当前会话能同步更新或清理。
- 会话列表在接口失败时有可见错误态，在结果为空时有可见空态，并且存在真实可点击的全局收藏占位入口。
- 代码注释与知识库说明均为简体中文。
