# 技术设计: Android 客户端验收问题修复

## 技术方案
### 核心技术
- Kotlin
- Jetpack Compose
- Hilt
- OkHttp WebSocket
- Retrofit

### 实现要点
- 在身份模块补齐 `updateIdentity` / `deleteIdentity` 的 Repository、ViewModel 和最小 UI
- 在会话列表模块增加明确空态、错误态与全局收藏入口占位
- 在 WebSocket 客户端增加协议码目录、结构化消息解析、`code=-4` / `code=-3` forceout 处理与真实自动重连
- 在聊天页消费结构化消息结果，并同步修正文档与任务状态

## 架构决策 ADR
### ADR-20260313-02: 验收问题优先以“修实现”为主，必要时同步修正文档状态
**上下文:** Claude 验收指出多个 `[√]` 与代码事实不一致。
**决策:** 对主链路缺口直接补齐实现；对暂不展开的能力继续保留 `[-]` 并补充边界说明。
**理由:** 优先提升代码与方案包的一致性，减少“虚标完成”带来的后续维护风险。
**替代方案:** 仅下调任务状态 → 拒绝原因: 会遗漏本轮本可快速补齐的核心能力。
**影响:** 会新增少量 UI 与状态逻辑，但能显著提高方案包可信度。
