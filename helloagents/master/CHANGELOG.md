# Changelog

本文件记录项目所有重要变更。
格式基于 Keep a Changelog，版本号遵循语义化版本。

## [Unreleased]

### 新增
- 初始化当前 `helloagents/` 知识库核心文档，覆盖项目概览、架构、API、数据模型、模块规范与历史索引。
- 新增 MT Photos 上游 API 文档快照与原始 OpenAPI JSON，整理 `https://mtmt.tech/api/` 的 OpenAPI 入口、认证方式、全量端点和 schema。
- WebSocket 匹配成功但未聊天的用户现在会写入本地 `chat_user_archive`，刷新历史列表时可从归档追加恢复。
- 新增媒体预览视频操作逻辑分析文档，覆盖播放/暂停、快退/快进、倍速/慢放、保存当前帧、创建抽帧任务入口和后续 UI 调整建议。
- 新增 `scripts/probe_upstream_ws.go` 手动诊断脚本，用于探测上游 WebSocket 在指定超时窗口内是否断开。
- 新增跨身份联系人接入能力：当前身份可从其他身份的历史、收藏和本地归档候选中选择用户并创建临时会话。
- 新增媒体尺寸持久化字段和 `/api/repairMediaDimensions` 历史回填接口，用于修复移动端瀑布流缺少宽高导致的单侧空白。
- 新增 `GET /api/chat/archiveSearch` 全局聊天归档搜索接口，支持不按身份过滤地检索 `target_user_id` 和归档快照中的用户名称。
- 聊天侧边栏新增“归档搜索”入口，可按用户 ID 或归档名称搜索所有身份下的本地归档会话并临时接入聊天。
- Android 会话列表新增“归档搜索”入口，接入 `/api/chat/archiveSearch` 并支持从搜索结果临时进入当前身份聊天。
- Android 会话列表新增“跨身份”入口，接入 `/api/chat/contactCandidates`，支持从其它身份的历史、收藏和归档候选临时进入当前身份聊天。
- Android 会话列表新增会话项“清未读”和“删除”动作，其中删除对齐 Web 端 `/api/deleteUpstreamUser`，成功后清理本地会话和消息缓存。
- Android 会话列表新增会话项“全局收藏 / 取消全局收藏”动作，对齐 Web 端 `/api/favorite/add` 与 `/api/favorite/remove`。
- Android 会话列表新增会话项“查在线”动作，对齐 Web 端 `ShowUserLoginInfo` WebSocket 协议并展示在线/离线/未知状态和最后时间。

### 变更
- 知识库以当前 Go 后端、Vue 前端、Android 客户端、SQL 迁移脚本和 Docker 构建为准；旧备份目录仅作为历史参考。
- mtPhoto 上游接入从账号密码登录/`jwt`/Cookie 授权码迁移为 `MTPHOTO_API_KEY`、`x-api-key` 与媒体 URL `auth_code` query。
- Web 前端聊天列表增加当前身份归属检查，切换身份后不复用旧身份的历史/收藏列表状态。
- 后端上游 WebSocket 客户端取消主动 ping，避免部分上游节点在收到 ping 后 reset 连接。
- 2026-05-10 执行 `~init` 知识库审计，刷新大型项目统计口径并补齐上游收藏代理 API 清单。
- 聊天消息列表中的视频恢复统一媒体预览入口，进入 `MediaPreview` 后可使用播放控制、倍速、保存当前帧和创建抽帧任务入口。
- 媒体预览视频保存当前帧会清理待处理 tap、暂停后同步播放状态，并在全屏保存 loading 期间保持浮层控制可见。
- 视频抽帧前端加强来源解析、探测成功门禁、续跑参数校验和帧分页 `seq` 去重。
- 媒体预览视频操作收敛为“视频工具”菜单，统一使用“保存当前帧”和“创建抽帧任务”文案；mtPhoto、全站媒体库、抖音下载和抽帧源视频预览按场景展示主动作。
- 聊天消息前端缓存改为按 `{当前身份ID}:{目标用户ID}` 隔离，避免多个身份联系同一用户时串消息、串乐观发送状态或串回显合并。
- `chat_user_archive` 归档搜索结果会返回 `ownerUserId`，用于区分同一目标用户在不同身份下的归档来源；返回快照继续过滤 cookie、token、password、secret 等敏感字段。
- 全局归档搜索结果改为展示所属身份名称和身份编码，便于区分目标用户来自哪个身份下的归档。
- PC 端“匹配新用户”改为主按钮单击单次匹配、右侧独立下拉按钮选择连续匹配 3/5/10 次；触屏端继续保留长按入口。
- 已收藏抖音用户再次解析作品后，会自动把最新媒体信息合并到后端收藏用户作品表，覆盖可能变化的媒体路径快照。

### 修复
- 跨身份联系人候选顺序改为保留来源身份历史/收藏的原始返回顺序，本地归档仅补充未返回联系人。
- 全站媒体库瀑布流现在读取持久化宽高并向前端传递，移动端两列布局在横竖图混排时更稳定。
- 修复全局收藏中跨身份“切换身份并聊天”依赖固定延时导致聊天室不显示对方身份或历史记录的问题。
- Android 普通身份选择现在会在新身份保存前清理旧本地聊天缓存，避免会话列表短暂展示上一身份数据。
- Android 应用级会话协调器新增非空身份 ID 变化检测，作为身份切换入口之外的聊天缓存隔离防线。
- Android 全局收藏跨身份“切换身份并聊天”现在会在切换成功后清理旧本地聊天缓存，并预置目标收藏对象会话，避免进入聊天页时沿用旧身份会话或消息状态。
