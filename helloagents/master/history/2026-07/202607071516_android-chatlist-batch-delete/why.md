# 变更提案: Android 会话列表批量删除

## 需求背景
Vue 会话侧边栏已支持选择模式和批量删除会话用户：用户可以进入选择模式、选择多个会话、调用 `/api/batchDeleteUpstreamUsers` 分批删除，并对部分失败保留失败项。Android 端当前只具备单个会话删除动作，缺少与 Vue 对齐的多选批量管理能力。

## 产品分析

### 目标用户与场景
- **用户群体:** 使用 Android 客户端长期管理大量历史/收藏会话的用户。
- **使用场景:** 清理多个无效或过期会话，尤其是历史列表堆积较多时。
- **核心痛点:** 当前 Android 只能逐个删除，会导致重复确认和大量手动操作。

### 价值主张与成功指标
- **价值主张:** 对齐 Vue 会话侧边栏的批量管理能力，降低移动端清理成本。
- **成功指标:** Android 会话列表可进入选择模式、批量选择、调用批量删除接口，并正确处理全成功和部分失败。

### 人文关怀
批量删除是破坏性操作，必须有确认弹窗，且部分失败时保留失败项，避免用户误以为全部删除成功。

## 变更内容
1. Android 网络层接入 `POST /api/batchDeleteUpstreamUsers`。
2. Android 会话列表增加选择模式、单项勾选、全选/退出和批量删除确认。
3. 批量删除成功后清理本地会话和消息缓存；部分失败时仅清理成功项并保留失败项选中。

## 范围边界
- **范围内:** Android 会话列表选择模式、批量删除 API、批量删除状态和 UI。
- **范围外:** 服务端接口改造、Vue 端变更、按日期批量删除、归档-only 特殊筛选 UI 重构。
- **拆分说明:** 本方案是 Android 与 Vue 会话侧边栏对齐中的批量管理切片，可独立实现和验证。

## 影响范围
- **模块:** Android Client
- **文件:** `NetworkModels.kt`、`NetworkStack.kt`、`LocalDatabase.kt`、`ChatListFeature.kt`、相关测试与知识库。
- **API:** 新增 Android 调用 `POST /api/batchDeleteUpstreamUsers`，请求 JSON `{ myUserId, userToIds }`。
- **数据:** 不新增数据库字段；成功删除时删除 `conversation_cache` 目标会话与 `message_cache` 对应聊天记录。

## 核心场景

### 需求: 会话列表批量删除
**模块:** Android Client
Android 会话列表应提供与 Vue 侧边栏一致的选择模式和批量删除能力。

#### 场景: 批量删除全成功
用户进入选择模式，选择多个会话并确认批量删除。
- Android 调用 `POST /api/batchDeleteUpstreamUsers`。
- 远端返回 `code=0` 且无失败项后，删除本地成功项会话和消息缓存。
- 退出选择模式并显示成功数量。

#### 场景: 批量删除部分失败
远端返回 `failedItems`。
- 仅删除成功项本地缓存。
- 保留失败项选中，继续停留在选择模式。
- 显示成功和失败数量。

#### 场景: 空选择或身份缺失
未选择任何会话或当前身份缺失。
- 不调用远端接口。
- 向用户显示失败原因或保持无操作。

## 风险评估
- **风险:** 批量删除属于破坏性操作，误删影响较大。
- **缓解:** 保留确认弹窗；本地缓存仅在远端成功后清理；部分失败不清理失败项。
