# 变更提案: Android 会话列表查询在线状态

## 需求背景
Vue 会话侧边栏已提供“查在线”动作：点击会话用户后通过 WebSocket 发送 `ShowUserLoginInfo`，收到 `code=30` 回包后展示对方是否在线和最后在线时间。Android 端当前 WebSocket 已能发送该指令，聊天页也有基础用户信息查询，但会话列表缺少入口、状态展示和结构化在线状态解析，导致会话列表管理能力与 Vue 端不一致。

## 产品分析

### 目标用户与场景
- **用户群体:** 使用 Android 客户端管理历史会话和收藏会话的操作用户。
- **使用场景:** 在会话列表中判断某个对象是否在线，再决定是否进入聊天。
- **核心痛点:** 当前必须进入聊天页或依赖原始提示，无法在列表页直接查看在线状态。

### 价值主张与成功指标
- **价值主张:** 对齐 Vue 会话侧边栏的单项管理动作，减少无效进入聊天。
- **成功指标:** 会话列表项可发起在线状态查询，并在收到 `code=30` 后展示在线/离线/未知与最后时间。

### 人文关怀
在线状态只展示服务端已返回的状态与时间，不新增持久化记录，不扩大隐私数据留存范围。

## 变更内容
1. 扩展 Android WebSocket `OnlineStatus` 事件，解析 `data.IF_Online` 与 `data.TimeAll`。
2. 在 Android 会话列表项增加“查在线”动作和在线状态弹窗。
3. 为发送失败、缺失字段和 UI 回调补充测试。

## 范围边界
- **范围内:** Android 会话列表“查在线”入口、WebSocket 在线状态结构化解析、ViewModel 状态和弹窗展示。
- **范围外:** 全量 Vue 能力补齐、聊天页在线状态 UI 重构、服务端协议修改、在线状态本地持久化。
- **拆分说明:** 本方案是 Android 与 Vue 能力对齐中的一个会话列表动作切片，可独立实现和验证。

## 影响范围
- **模块:** Android Client
- **文件:** `core/websocket/LiaoWebSocketClient.kt`、`feature/chatlist/ChatListFeature.kt`、相关测试与知识库。
- **API:** 不新增 HTTP API；复用 WebSocket `ShowUserLoginInfo` 与 `code=30`。
- **数据:** 不新增数据库字段，不持久化在线状态。

## 核心场景

### 需求: 会话列表查询在线状态
**模块:** Android Client
Android 会话列表应提供与 Vue 侧边栏一致的“查在线”动作。

#### 场景: 查询在线用户
用户点击某个会话项的“查在线”按钮。
- Android 通过 WebSocket 发送 `ShowUserLoginInfo`，`id` 为当前身份 ID，`msg` 为目标用户 ID。
- 收到 `code=30` 且 `data.IF_Online = "1"` 时，展示目标用户在线。
- 展示 `data.TimeAll` 作为最后时间；字段缺失时显示兜底文案。

#### 场景: 查询离线或未知用户
服务端返回 `data.IF_Online = "0"` 或缺失/异常值。
- `"0"` 展示离线。
- 缺失或异常值展示状态未知，不误判为在线。

#### 场景: WebSocket 不可用
当前身份缺失或 WebSocket 未连接。
- 不展示在线状态弹窗。
- 会话列表提示失败原因。

## 风险评估
- **风险:** `code=30` 回包字段可能缺失或结构变化，导致误展示。
- **缓解:** 在线状态解析使用可空布尔值，未知状态单独展示；测试覆盖在线、离线和缺失字段。
