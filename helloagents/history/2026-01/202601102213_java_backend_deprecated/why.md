# 变更提案: 标记 Java 版本为弃用（仅参考）

## 需求背景
当前项目后端以 Go 版本为主（`cmd/liao/` + `internal/`），仓库中保留的 Java(Spring Boot) 实现仅用于参考。
为避免误用与误导，需要在 Java 版本目录中显式标记其已不可用/不再维护。

## 变更内容
1. 在 Java 版本目录新增弃用说明文件，明确其“仅参考、不可运行/不维护”的定位，并指向 Go 版本入口。
2. 同步更新与工程说明相关的文档/知识库，避免继续将 Java 版本描述为当前后端实现。

## 影响范围
- **模块:** 文档 / 工程结构说明
- **文件:**
  - `src/`（Java 版本目录标记）
  - `CLAUDE.md`（开发助手指引）
  - `helloagents/wiki/*`（SSOT 知识库同步）
- **API:** 无
- **数据:** 无

## 核心场景

### 需求: Java 版本弃用标记
**模块:** 工程结构/文档
当开发者浏览仓库结构或尝试运行后端时，应能快速识别 Java 目录为“仅参考、不可用”，并明确应使用 Go 版本后端入口。

#### 场景: 开发者查看仓库结构
在查看 `src/` 或 `src/main/java/` 时能看到弃用说明。
- 预期结果: 明确提示 Java 版本已弃用，仅供参考
- 预期结果: 提供 Go 版本入口与运行方式的指引链接/路径

## 风险评估
- **风险:** 文档标记不一致导致开发者仍按 Java 流程尝试运行
- **缓解:** 在 Java 目录与开发助手指引中同时标注，并在知识库同步“后端以 Go 为准”
