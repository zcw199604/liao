# 媒体预览视频操作逻辑分析与调整建议

日期: 2026-05-10
范围: Web 前端 `MediaPreview` 视频预览、抓帧、抽帧入口、抽帧任务中心与相关入口路径
结论来源: 主代理本地只读分析 + 3 个只读子代理并行分析

---

## 1. 目标

本文件整理当前媒体预览中视频操作逻辑，并给出后续 UI 调整建议。关注点包括:

- 视频播放、暂停、快进、快退、慢放/倍速。
- 当前帧抓取与异步抽帧入口。
- 全屏、手势、Plyr 与原生 `video` 状态同步。
- 聊天、抖音下载、抽帧任务中心等入口的一致性。

2026-05-10 后续 `~exec` 已执行第一批风险收敛实现，覆盖聊天视频入口、全屏抓帧状态、Plyr/原生状态同步、抽帧来源/探测/续跑门禁和帧分页去重。完整播放器 UI 重组、移动端数值浮层和 mtPhoto 任务源视频扩展仍属于后续独立优化。

## 2. 当前用户路径

### 2.1 统一预览组件

核心组件是 `frontend/src/components/media/MediaPreview.vue`。它支持单媒体兼容模式和 `mediaList` 画廊模式:

- `type=image`: 图片缩放、拖动、左右切换、Live Photo 动态预览。
- `type=video`: Plyr 播放器、浮层控制、倍速、抓帧、抽帧。
- `type=file`: 仅展示文件占位和下载。

### 2.2 进入视频预览的路径

| 来源 | 当前行为 | 风险 |
|------|----------|------|
| `MessageBubble.vue` | 图片/视频点击都会 dispatch `preview-media` | 支持统一视频预览 |
| `MessageList.vue` | 图片和视频都通过 `ChatMedia` 预览并 dispatch `preview-media` | 已统一进入媒体预览 |
| `DouyinDownloadModal.vue` | 构造 `previewUrl/previewType/previewMediaList` 后打开 `MediaPreview` | 路径完整 |
| `VideoExtractCreateModal.vue` | 创建抽帧前可预览源视频 | 可复用倍速/抓帧 |
| `VideoExtractTaskModal.vue` | 可预览 upload 源视频；帧图点击进入图片预览 | mtPhoto 任务源视频暂不可预览 |

聊天视频入口已统一；后续重点转向播放器工具栏布局、移动端反馈和更细的控件误触边界。

## 3. 当前视频操作逻辑

### 3.1 播放器基础

`MediaPreview.vue` 中视频元素使用:

- `autoplay`
- `controls`
- `playsinline`
- `webkit-playsinline`
- `poster`

组件初始化后会用 Plyr 接管播放器控制栏。Plyr 配置移除了中央大播放按钮，并禁用 `clickToPlay`，由组件自己的点击/手势逻辑处理播放暂停。

播放状态通过原生 `video` 的 `play/pause/ended` 事件同步到 `isVideoPlaying`。当前实现会同时写入 Plyr 与原生 `video` 的倍速、进度和音量，降低抓帧、横滑 seek、慢放和长按 2x 的状态偏差。

### 3.2 播放/暂停

当前有三种触发方式:

- 原生/Plyr 控制栏。
- 单击视频区域: 先显示浮层，再延迟 300ms 执行播放或暂停，以便识别双击。
- 浮层中央按钮: 立即播放/暂停，并清理待处理的单击动作，避免状态被翻转两次。

风险:

- 单击延迟期间如果用户又操作原生/Plyr 控制栏，待处理的单击可能按旧意图执行。
- 抓帧时会清理待处理 tap、暂停视频并主动同步播放状态；全屏抓帧 loading 期间浮层工具栏保持可见。

### 3.3 快进/快退

当前有两种触发方式:

- 浮层左右按钮: 每次倒退/快进 1 秒。
- 视频区域横向滑动: 按视频宽度换算步数，每步 1 秒。

注意:

- 键盘 `ArrowLeft/ArrowRight` 当前用于切换上一项/下一项媒体，不用于视频 seek。
- 横滑 seek 没有数值反馈，用户只能通过画面或控制栏判断跳转幅度。

### 3.4 慢放/倍速

倍速按钮出现在:

- 非全屏: 预览层顶部工具栏。
- 全屏: 视频左上角。

固定倍速选项:

- `0.1`
- `0.25`
- `0.5`
- `1`
- `1.5`
- `2`
- `5`

选择后会写入 `localStorage.media_preview_playback_rate`，下次打开继续沿用。

长按倍速按钮超过 320ms 会临时进入 2x:

- 原本播放: 松开后恢复用户选择的倍速。
- 原本暂停: 长按时临时播放，松开后恢复暂停。

### 3.5 手势

当前视频区域支持:

- 单击: 播放/暂停。
- 双击: 切换全屏。
- 横滑: 快退/快进。
- 竖滑: 调整音量。

Plyr 控制栏区域会被排除，避免手势抢事件。若 Plyr 初始化失败或初始化前仍显示原生 controls，原生控制栏区域不一定能被当前选择器排除。

### 3.6 抓帧

抓帧按钮位置:

- 非全屏: 底部操作栏常驻显示。
- 全屏: 右侧按钮组，随浮层显示/隐藏。

抓帧流程:

1. 如果视频正在播放，先暂停。
2. 检查 `readyState >= 2`、`videoWidth/videoHeight` 可用。
3. 用 canvas 绘制当前帧。
4. 直接下载 PNG。
5. 若当前有身份，则上传到图片库；来源可带 `local/mtphoto/douyin` 元数据。

跨域视频可能触发 canvas 安全限制，此时提示用户改用本地库或抽帧功能。

主要风险:

- 全屏抓帧按钮依赖 1 秒自动隐藏的浮层；抓帧耗时超过 1 秒时，loading 反馈会消失。
- 抓帧暂停后建议主动清理待处理 tap，并立即同步 `isVideoPlaying`。

## 4. 当前抽帧链路

### 4.1 入口条件

`MediaPreview.vue` 的 `canExtractFrames` 只在当前媒体是视频时计算，满足以下任一来源时显示入口:

- 带 `md5` 且 URL 是 `/lsp/` 或 `/api/`，按 mtPhoto 来源处理。
- URL 是 `/videos/`。
- URL 是 `/upload/`。
- URL 中包含 `/upload/`。

点击后调用:

```ts
videoExtractStore.openCreateFromMedia(currentMedia, userStore.currentUser?.id)
```

成功后关闭当前预览，打开抽帧创建弹窗。

### 4.2 创建任务

`VideoExtractCreateModal.vue` 负责:

- 展示 ffprobe 探测结果。
- 配置抽帧模式:
  - 关键帧
  - 固定 FPS
  - 每一帧
- 配置时间区间、最大帧数、输出格式、JPG 质量。
- 预览源视频。
- 创建任务后打开任务中心。

风险:

- 当前创建按钮未强制要求 `probe` 成功，探测失败后仍可能提交。
- 入口条件与 store 中来源解析可能不完全一致，尤其是 `/videos/` 是否能被 `extractUploadLocalPath` 转成后端需要的 `localPath` 需要确认。

### 4.3 任务中心

`VideoExtractTaskModal.vue` 负责:

- 任务列表与状态展示。
- 运行中任务终止。
- `PAUSED_LIMIT` 或 `PAUSED_USER` 状态下继续抽帧。
- 帧图结果虚拟滚动/瀑布流预览。
- 实时日志展示。
- upload 源视频预览。

风险:

- 继续抽帧的前端校验较弱，空提交、`endSec <= cursorOutTimeSec`、`maxFrames <= framesExtracted` 主要依赖后端兜底。
- mtPhoto 任务详情页没有源视频预览入口。
- 帧图预览只包含当前已加载页，不能在预览内自然浏览全部结果。
- 帧分页追加缺少前端去重保护。

## 5. 主要问题清单

### P0: 入口能力不一致

聊天视频在 `MessageBubble.vue` 能进入统一预览，但 `MessageList.vue` 的视频预览被关闭。用户会遇到“同样是聊天视频，有时能快进/暂停/抽帧，有时不能”的体验。

建议:

- 统一聊天视频点击行为，默认进入 `MediaPreview`。
- 若出于性能考虑不自动预览，应给视频缩略图一个显式播放/预览按钮。

### P1: 全屏抓帧反馈可能消失

全屏抓帧按钮跟随浮层显示，浮层 1 秒后隐藏。抓帧期间按钮 loading 可能被隐藏。

建议:

- 抓帧期间保持浮层可见。
- 或让全屏右侧“抓帧/抽帧”工具栏独立于浮层自动隐藏。

### P1: Plyr 与原生 video 状态需要收敛

当前倍速、seek、音量优先写 Plyr 后直接返回。若 Plyr 写入和原生 video 属性短暂不同步，抓帧、连续 seek、音量探测可能读到旧状态。

建议:

- 封装统一播放器适配层。
- 写 Plyr 后同步校验原生 `video.currentTime/playbackRate/volume`。
- 抓帧前以 Plyr 当前时间为准落回原生 video，再绘制 canvas。

### P1: 抽帧创建前缺少强探测门禁

文档约束是先探测源视频元数据，但 UI 未强制 `probe` 成功才允许创建。

建议:

- 创建按钮禁用条件加入 `!probe || !!probeError`。
- 或提交前重新探测，失败则阻断创建。

### P2: 继续抽帧前端校验不足

继续抽帧允许空提交，且没有前端校验新的边界是否真正扩大。

建议:

- 空提交提示“请填写新的 endSec 或 maxFrames”。
- 校验 `endSec > cursorOutTimeSec`。
- 校验 `maxFrames > framesExtracted`。

### P2: 手势与控制栏边界不够稳

Plyr 控制栏会被排除，但原生 controls 降级时不一定能排除。

建议:

- Plyr 初始化失败时禁用自定义点击/滑动手势。
- 或增加对原生 controls 区域的命中保护。

### P2: 可访问性不足

多个图标按钮主要依赖 `title`，部分可点击容器缺少键盘语义。

建议:

- 图标按钮补 `aria-label`。
- 可点击非按钮元素补 `role=button`、`tabindex=0`、Enter/Space 触发，或改成真实 button。

## 6. 建议调整方案

### 6.1 最小修正方案

适合先快速降低风险，不大改交互模型。

1. 统一聊天视频进入 `MediaPreview` 的入口。
2. 抓帧开始时清理待处理 tap，暂停后立即同步播放状态。
3. 全屏抓帧期间保持工具栏可见。
4. 对 Plyr 写入后的关键状态做原生 video 同步。
5. 统一 `canExtractFrames` 与 `openCreateFromMedia` 的来源解析。
6. 创建抽帧任务前强制探测成功。
7. 继续抽帧补前端校验。
8. 帧分页合并按 `seq` 去重。

### 6.2 UI 重组方案

适合后续做一轮更完整的体验优化。

1. 把视频操作拆成三层:
   - 播放层: Plyr 控制栏和中央浮层播放/暂停。
   - 精准操作层: 快退/快进、倍速、抓帧、抽帧。
   - 画廊层: 上一项/下一项媒体、缩略图、下载、详情。
2. 全屏模式固定保留右侧工具栏:
   - 抓帧。
   - 抽帧。
   - 信息。
   - 下载。
3. 横滑/竖滑显示轻量反馈:
   - `+3s` / `-2s`
   - `音量 60%`
4. 明确键盘语义:
   - `ArrowLeft/ArrowRight` 当前是切换媒体。
   - 如要支持视频 seek，应改为在视频聚焦时 seek，其他区域切换媒体。

## 7. 验收清单

### 播放控制

- 单击视频只触发一次播放/暂停。
- 点击浮层按钮不会被延迟 tap 再次翻转状态。
- 双击进入/退出全屏稳定。
- Plyr 控制栏操作不触发外层手势。

### 快进/快退

- 浮层按钮每次准确前后 1 秒。
- 横滑按 1 秒步进，边界不越过 0 或 duration。
- 多媒体画廊中左右箭头行为符合产品决策。

### 倍速/慢放

- 选择倍速后立即生效并持久化。
- 0.1x/0.25x/0.5x 慢放在 Plyr 和原生 video 上状态一致。
- 长按 2x 松开后恢复原倍速。
- 原本暂停时长按 2x 松开后恢复暂停。

### 抓帧

- 抓帧前自动暂停并同步 UI。
- 全屏抓帧期间 loading 可见。
- 当前帧文件名包含时间戳。
- 跨域失败提示明确。
- 登录身份存在时上传到图片库；无身份时只下载并提示。

### 抽帧

- 只有可解析为 upload localPath 或 mtPhoto md5 的视频显示抽帧入口。
- 探测失败不能创建任务。
- 创建后自动进入任务中心并选中新任务。
- 终止任务后可继续，继续参数边界校验清晰。
- 帧预览不重复，分页加载行为稳定。

### 入口一致性

- 聊天视频、抖音视频、上传视频、抽帧源视频都能进入统一预览能力。
- 不支持抽帧的视频不会展示可点击入口，或点击后给出明确原因。

## 8. 后续实现建议

建议拆成两批:

### 第一批: 风险收敛

- 修复聊天视频入口不一致。
- 修复全屏抓帧反馈。
- 修复抓帧状态同步。
- 抽帧入口和创建探测门禁收敛。
- 继续抽帧前端校验。

### 第二批: 体验增强

- 横滑/竖滑反馈。
- 全屏工具栏重组。
- 键盘语义细化。
- 可访问性补强。
- 帧预览中加载更多或明确提示当前只浏览已加载帧。
