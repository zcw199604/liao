视频抽帧操作界面优化方案

基于当前 Vue 抽帧创建弹窗、任务中心和 3 个子代理只读分析结果，整理 3 个可选择的操作示例。

创建流程 任务中心 移动端体验
参数暴露过早 I 帧、场景阈值、FPS、JPG 质量同时出现，普通操作需要先理解专业概念。
入口闭环完整 媒体预览、设置抽屉和任务中心上传都能进入抽帧，但缺少按目标推荐的路径。
任务中心过载 列表、上传、详情、终止、续跑、删除、预览和日志挤在一个 Modal。
状态操作可再明确 续跑应从“填写绝对 endSec/maxFrames”改为“追加多少秒/多少帧”。

选择建议

方案 优先级 最小落地范围 主要风险
方案一：预设向导 高 优先改创建弹窗，保留现有任务中心和 API。 需要定义预设到现有参数的映射，并补组件测试。
方案二：任务工作台 中 增强任务中心列表、续跑、删除确认、状态筛选。 批量操作和失败重试可能需要后端补幂等接口。
方案三：一体化工作台 中低 新建工作台式入口，逐步替代创建弹窗和任务中心。 体验最好但改动最大，需重新验证移动端滚动和状态流转。

建议先选方案一作为低风险迭代；如果你更关注大量任务管理，再选方案二；如果愿意做一次较大的体验重构，再选方案三。
